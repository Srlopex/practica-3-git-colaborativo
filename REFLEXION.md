# Reflexión grupal

## ¿Qué ventajas tiene trabajar con ramas?
Trabajar con ramas permite desarrollar nuevas funcionalidades sin afectar la rama principal. Cada miembro puede trabajar de forma independiente y luego integrar los cambios.

## ¿Por qué son útiles los Pull Requests en un equipo?
Los Pull Requests permiten revisar los cambios antes de integrarlos, facilitando la colaboración y evitando errores en el código principal.

## ¿Qué aprendisteis al resolver el conflicto?
Aprendimos cómo Git detecta conflictos cuando dos ramas modifican la misma parte del archivo y cómo resolverlos manualmente eligiendo el contenido correcto.

## Dificultades encontradas durante la práctica
Tuvimos algunas dificultades al usar comandos de Git y al resolver conflictos, pero entendimos mejor el flujo de trabajo colaborativo.
